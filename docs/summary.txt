DOCKLENS — Scanner de Sécurité pour Images Docker

## OBJECTIF

Fournir un outil DevSecOps capable de scanner les images Docker (si possible les clusters Kubernetes) pour détecter les vulnérabilités connues (CVE + EUVD) dans les paquets systèmes et applicatifs, sans exécuter les conteneurs, et afficher les résultats directement dans le terminal.

## FONCTIONNALITÉS

* Scanner les images locales et Kubernetes sans lancement.
* Extraire un SBOM minimal (Software Bill of Materials).
* Identifier les vulnérabilités CVE / EUVD (CVE européennes).
* Prioriser les failles selon CVSS, exposition et ancienneté.
* Proposer des liens de correction de vulnérabilités.
* Afficher les résultats lisiblement dans le terminal (tableaux Rich).

## PÉRIMÈTRE

MODE A — LOCAL

Cible :

* Images Docker présentes sur la machine.

Entrée :

* image:tag
* éventuellement Dockerfile

Sortie :

* SBOM
* CVE/EUVD détectées
* score de risque

Interface :

* CLI/TUI avec menus interactifs
* Typer + Rich + InquirerPy

MODE B — KUBERNETES (LECTURE SEULE)

Cible :

* Images référencées dans les workloads :

  * Pods
  * Deployments
  * DaemonSets
  * etc.

Accès :

* In-cluster : ServiceAccount avec RBAC read-only
* Off-cluster : via kubeconfig (~/.kube/config)

Fonctionnement :

* Énumération des images du cluster
* Scan local ou via accès registry
* Affichage direct des résultats dans le terminal
